# Ancient-Type-DB (고활자 정보 관리 시스템)

본 프로젝트는 고문헌(예: 월인천강지곡)의 스캔 이미지에서 추출된 활자 정보를 체계적으로 관리하고, 3D 모델(STL) 및 이미지 데이터를 시각화하는 **MFC 기반의 데스크톱 애플리케이션**입니다.

---

## 🚀 주요 기능

### 1. 데이터 시각화 및 인터랙션
* **고해상도 스캔 데이터 로드**: 각 장(Sheet)별 고해상도 이미지를 로드하여 화면에 표시합니다.
* **활자 위치 표시**: `typeDB.csv`의 좌표 데이터를 기반으로 이미지 상의 모든 활자 위치에 녹색 박스를 생성하며, 현재 선택된 활자는 빨간색 박스로 강조합니다.
* **마우스 인터랙션**: 책 이미지 위를 직접 클릭하여 해당 위치의 활자 정보를 즉시 확인할 수 있습니다.

### 2. 3D 활자 모형 뷰어 (VTK 통합)
* **STL 로드**: 고활자의 실제 모양을 본뜬 3D STL 파일을 화면에 렌더링합니다.
* **트랙볼 카메라 조작**: 마우스 드래그를 통해 활자 모형을 자유롭게 회전하고 확대/축소하여 관찰할 수 있는 트랙볼 모드를 적용했습니다.
* **디자인 최적화**: 퐁(Phong) 셰이딩과 금속성 질감 표현을 통해 실제 활자와 유사한 시각 효과를 제공합니다.

### 3. 고효율 데이터 관리
* **CSV 기반 DB**: 수천 개의 활자 정보를 CSV 파일로 관리하며, 로딩 시 실시간으로 전체 글자 수, 종 수 등 통계 데이터를 산출합니다.
* **계층적 탐색**: 책 이름 -> 장(Sheet) -> 개별 활자로 이어지는 직관적인 탐색 환경을 제공합니다.
* **구성 글자 리스트**: 동일한 글자가 나타나는 모든 위치를 리스트로 보여주며, 클릭 시 장 이동 없이도 모형을 빠르게 비교할 수 있습니다.

### 4. 기술적 최적화 (UX 개선)
* **깜빡임 제거 (Flicker-Free)**: `WM_ERASEBKGND` 제어 및 부분 갱신(`Invalidate(FALSE)`) 기법을 적용하여 잦은 데이터 변경에도 눈의 피로가 없는 부드러운 화면 전환을 구현했습니다.
* **뷰 독립성 유지**: 리스트나 활자 정보 변경 시 사용자가 보고 있는 현재 페이지(장)를 고정하여 작업의 연속성을 보장합니다.

## 🛠 기술 스택
* **Language**: C++
* **Framework**: MFC (Microsoft Foundation Class)
* **Library**: VTK (Visualization Toolkit) 9.x
* **Version Control**: Git

---

## 📂 프로젝트 구조
* `01_scan/`: 고문헌 원본 스캔 이미지 (JPG)
* `03_type/`: 개별 활자 추출 이미지
* `04_3d/`: 활자 3D 모델 파일 (STL)
* `typeDB.csv`: 활자 유니코드, 위치 좌표(x, y, w, h), 장/행/번 정보 포함

## ⚙️ 설정 방법 (라이브러리 경로)
본 프로젝트를 빌드하기 위해서는 Visual Studio 프로젝트 속성에서 VTK 라이브러리 설정이 필요합니다.
1. **VC++ 디렉터리**: VTK `include` 및 `lib` 경로 추가
2. **링커 -> 입력**: 사용 중인 VTK 모듈 `.lib` 파일 목록 추가
3. **DLL**: 실행 파일 위치에 필요한 VTK DLL 파일 배치
